# Agent CLI DevKit システム仕様

## 1. 概要

Agent CLI DevKit は、AI エージェント CLI（Claude Code / Codex CLI / Grok CLI など）の開発環境を管理する Electron 製デスクトップアプリケーションである。
Claude Developer Tool と Codex Developer Tool の全機能を統合し、さらに Grok CLI（Grok Build）対応を追加したもので、UI・内部構造ともに「agent（Claude / Codex / Grok / 今後追加される CLI）」を単位としてグループ化されている。

- プロジェクト名: `agent_cli_devkit`
- 設定ファイルディレクトリ: `~/.agent_cli_devkit`（`src/shared/constants.ts` の `APP_DIR_NAME`。現状アプリ自体は永続設定を持たないため予約）
- 実行ファイル名: `agent_cli_devkit`
- 配布ファイル名: `agent-cli-devkit-<version>-<os>-<arch>`
- renderer へ公開するグローバル API: `window.agentCliDevkit`

## 2. アーキテクチャ

### 2.1 設計原則

1. **agent 別ファイル分離** — Claude 系と Codex 系の実装が同一ファイルに混在しないよう、層ごとに `claude/` / `codex/` ディレクトリへ分割する。両者を同時に参照してよいのは「登録ポイント」（registry・合成ファイル）のみ。
2. **共通化** — agent に依存しない処理（WSL 検出、ホームディレクトリのファイル操作、git 実行、サイズ計測、frontmatter/YAML/TOML 編集）は共通モジュールに置き、各 agent から利用する。
3. **拡張容易性** — 新しい agent CLI は「agent 別ディレクトリを追加し、各層の登録ポイントに 1 行追加する」だけで組み込める（§6 参照）。

### 2.2 ディレクトリ構成と責務

```text
src/
├── shared/
│   ├── constants.ts              # アプリ共通定数・共通 IPC チャンネル (window/system/updater)
│   ├── types.ts                  # アプリ共通型 (OSType / UpdateState)
│   ├── ipc.ts                    # window.agentCliDevkit の型 (agent API を束ねる登録ポイント)
│   └── agents/
│       ├── types.ts              # agent 非依存のドメイン型 (AgentEnvironment / MCP / Cleanup / Asset / Settings)
│       ├── claude/constants.ts   # Claude の定数・IPC チャンネル・レジストリ (クリーンアップ候補/設定項目)
│       ├── claude/types.ts       # Claude 固有型 + 共通型の再エクスポート
│       ├── claude/ipc.ts         # ClaudeIpcApi (window.agentCliDevkit.claude の型)
│       ├── codex/…               # Codex も同構成
│       └── grok/…                # Grok も同構成
├── main/
│   ├── index.ts                  # 起動・ウィンドウ生成 (sandbox 無効・contextIsolation 有効)
│   ├── ipc/
│   │   ├── index.ts              # 登録ポイント: WslDetector を生成し各 agent へ注入
│   │   ├── claude/               # Claude 系 IPC ハンドラ (desktop/code/cleanup/asset/settings)
│   │   ├── codex/                # Codex 系 IPC ハンドラ (mcp/asset/cleanup/config)
│   │   ├── grok/                 # Grok 系 IPC ハンドラ (mcp/asset/cleanup/config)
│   │   ├── systemHandlers.ts     # system:* (テーマ/ロケール/バージョン)
│   │   ├── windowHandlers.ts     # window:* (最小化/最大化/閉じる)
│   │   └── updater.ts            # updater:* (自動アップデート)
│   ├── services/
│   │   ├── claude/               # ClaudeDesktopManager / ClaudeCodeManager / ClaudeCleanupManager
│   │   │                         # / ClaudeAssetManager / ClaudeSettingsManager
│   │   ├── codex/                # CodexMcpManager / CodexAssetManager / CodexCleanupManager / CodexConfigManager
│   │   ├── grok/                 # GrokMcpManager / GrokAssetManager / GrokCleanupManager / GrokConfigManager
│   │   ├── common/
│   │   │   ├── wsl/WslDetector.ts  # WSL distro 検出 (agent 非依存、判定コマンドは呼び出し側が渡す)
│   │   │   ├── wsl/HomeFs.ts       # native/WSL のホーム配下ファイル操作の抽象化 (UNC/コマンドモード)
│   │   │   └── git/GitRunner.ts    # git clone/pull 実行
│   │   └── updater.ts            # electron-updater ラッパー
│   └── utils/                    # frontmatter / fsSize / yamlPreserve / tomlEdit / console-bridge
├── preload/
│   ├── index.ts                  # 登録ポイント: 共通 API + agent API を window.agentCliDevkit に公開
│   └── agents/claude.ts, codex.ts, grok.ts  # agent 別 IPC ブリッジ
└── renderer/
    ├── App.tsx                   # ルート定義 (registry から生成)
    ├── agents/
    │   ├── types.ts              # AgentModule / AgentFeature 型
    │   ├── index.ts              # 登録ポイント: AGENT_MODULES = [claudeModule, codexModule, grokModule]
    │   ├── claude/               # Claude 系画面 (index.tsx = モジュール定義 + 各コンポーネント)
    │   ├── codex/                # Codex 系画面
    │   └── grok/                 # Grok 系画面
    ├── components/               # 共通 UI (TitleBar / Dashboard / Layout / UpdateNotification)
    ├── i18n/locales/{ja,en}/     # app.ts (共通文言) + claude.ts + codex.ts + grok.ts + index.ts (合成)
    ├── store/useAppStore.ts      # テーマ・言語の zustand ストア
    └── utils/format.ts           # 表示整形 (サイズ/日時/環境キー)
```

### 2.3 プロセス構成と IPC

- メインプロセス起動時に `registerIpcHandlers()`（`src/main/ipc/index.ts`）が `WslDetector` を 1 つ生成し、Claude 系・Codex 系の各マネージャに注入して IPC ハンドラを登録する。
- preload は `sandbox: false`（`contextIsolation: true` は維持）で動作し、agent 別ブリッジファイル（`preload/agents/*.ts`）を require 合成して `window.agentCliDevkit` を公開する。sandbox を無効化しているのは preload を agent 別ファイルに分割するためで、shared の定数・型を preload から直接 import できる利点もある。
- IPC チャンネルは agent プレフィックスで名前空間化されている:
  - Claude: `claude-desktop:*` / `claude-code:*` / `claude-cleanup:*` / `claude-asset:*` / `claude-settings:*`
  - Codex: `codex-mcp:*` / `codex-asset:*` / `codex-cleanup:*` / `codex-config:*`
  - Grok: `grok-mcp:*` / `grok-asset:*` / `grok-cleanup:*` / `grok-config:*`
  - 共通: `window:*` / `system:*` / `updater:*` / `main:console`

### 2.4 renderer API（window.agentCliDevkit）

```text
window.agentCliDevkit
├── system   { getTheme, getLocale, getVersion }
├── window   { minimize, maximize, close, isMaximized }
├── updater  { getState, check, download, quitAndInstall, onStateChanged }
├── claude   { desktop, code, cleanup, asset, settings }   # ClaudeIpcApi
├── codex    { mcp, asset, cleanup, config }               # CodexIpcApi
└── grok     { mcp, asset, cleanup, config }               # GrokIpcApi
```

## 3. 機能仕様

### 3.1 Claude グループ

| 機能 | ルート | 内容 |
| --- | --- | --- |
| Claude Desktop MCP 管理 | `/claude/desktop-mcp` | `claude_desktop_config.json` の mcpServers を有効/無効/並べ替え。無効分は同ディレクトリの `claude_desktop_config_disabled.json` へ退避。Claude Desktop の再起動起動に対応。 |
| Claude Code MCP 管理 | `/claude/code-mcp` | `~/.claude.json` のトップレベル mcpServers を管理。無効分は `~/.claude-disabled-mcp.json`（sidecar）へ退避。native + Claude 導入済み WSL distro が対象。 |
| Claude Code Agent・Skill 管理 | `/claude/agent-skill` | `~/.claude/agents`（.md）・`~/.claude/skills`（フォルダ）の一覧/ZIP ダウンロード/ZIP・md アップロード/削除。Anthropic 公式スキル（anthropics/skills）のインポート。 |
| Claude Code 設定 | `/claude/settings` | `~/.claude/settings.json` の登録項目をテーブル編集・直接編集（関係ない項目には触れない差分マージ）。 |
| Claude Code クリーンアップ | `/claude/cleanup` | `~/.claude` 配下の履歴/キャッシュ/一時/ログ（projects・plans・file-history・history.jsonl・shell-snapshots・cache・debug・sessions・session-env・tasks・backups）を選択削除。「その他のツール」として Serena（~/.serena）の登録プロジェクト初期化・ログ削除に対応。 |

### 3.2 Codex グループ

| 機能 | ルート | 内容 |
| --- | --- | --- |
| Codex MCP 管理 | `/codex/mcp` | `~/.codex/config.toml` の `[mcp_servers.*]` を有効/無効/並べ替え（TOML の書式・コメントを保持）。無効分は `~/.codex/config-disabled-mcp.toml` へ退避。native + Codex 導入済み WSL distro が対象。 |
| Codex Agent・Skill 管理 | `/codex/agent-skill` | `~/.codex/agents`（.toml）・`~/.agents/skills`（フォルダ）の一覧/ZIP ダウンロード/ZIP・ファイルアップロード/削除。OpenAI 公式スキル（openai/skills、カタログは skills/.curated 配下）のインポート。 |
| Codex 設定 | `/codex/settings` | `~/.codex/config.toml` の登録項目をテーブル編集・直接編集（関係ない項目には触れない差分マージ）。 |
| Codex クリーンアップ | `/codex/cleanup` | `~/.codex` 配下（sessions・cache・generated_images・tmp・log・models_cache.json・session_index.jsonl）を選択削除。「その他のツール」（Serena）にも対応。 |

### 3.3 Grok グループ

対象は `~/.grok` 配下のみ（CLI 横断の共有ディレクトリ `~/.agents/skills` や、Grok が互換読み込みする `~/.claude` 系設定には触れない）。

| 機能 | ルート | 内容 |
| --- | --- | --- |
| Grok MCP 管理 | `/grok/mcp` | `~/.grok/config.toml` の `[mcp_servers.*]` を有効/無効/並べ替え（TOML の書式・コメントを保持）。無効分は `~/.grok/config-disabled-mcp.toml` へ退避。native + Grok 導入済み WSL distro が対象。 |
| Grok Agent・Skill 管理 | `/grok/agent-skill` | `~/.grok/agents`（直下の .md のみ・非再帰。Grok CLI の実挙動に準拠。サブディレクトリを含む ZIP のアップロードは孤児ファイル防止のため拒否）・`~/.grok/skills`（フォルダ）の一覧/ZIP ダウンロード/ZIP・md アップロード/削除。xAI 公式スキル（xai-org/plugin-marketplace、カタログは default-skills 配下）のインポート。 |
| Grok 設定 | `/grok/settings` | `~/.grok/config.toml` の登録項目をテーブル編集・直接編集（関係ない項目には触れない差分マージ）。 |
| Grok クリーンアップ | `/grok/cleanup` | `~/.grok` 配下（sessions・memory・debug・plans・docs・sandbox-events.jsonl）と `bin` 配下の旧バージョンバイナリ（`grok-<version>` 形式のうち最新以外。起動用 `grok` / `grok.exe` と最新バージョンは保護）を選択削除。「その他のツール」（Serena）にも対応。 |

### 3.4 共通機能

- **ダッシュボード** — agent ごとにグループ見出し（Claude / Codex / Grok）を表示し、配下に全機能のカードを並べる。カードはアイコンを左・テキストを右に置く横並びのコンパクトレイアウト。`renderer/agents/index.ts` の registry から自動生成。
- **タイトルバー** — ダッシュボードアイコンに続き、agent グループの代表ボタン（Claude / Codex / Grok）のみを表示するアコーディオン方式。代表ボタンをクリックするとそのグループの機能アイコンが展開され、別グループを展開すると先に開いていたグループは自動で閉じる（同じボタンの再クリックで閉じる）。画面遷移時は遷移先の機能が属するグループを自動展開する。バーガーメニューでは agent 名の小見出し付きで全機能を一覧。テーマ切替（ライト/ダーク）・言語切替（日本語/英語）・終了。
- **自動アップデート** — GitHub Releases を対象に electron-updater でチェック/ダウンロード/適用。
- **WSL 対応** — Windows では WSL distro を検出し、対象 CLI が導入された distro を各機能の管理対象として表示。ファイルアクセスは `\\wsl.localhost\` / `\\wsl$\` の UNC を優先し、不可なら wsl コマンド実行にフォールバック。
- **Agent・Skill の ZIP ダウンロード** — 既定ファイル名は `<agent>_<kind>[-<distro>].zip`（例: claude_skills.zip / grok_agents-Ubuntu-24.04.zip）。
- **Agent・Skill 一覧の参照列** — 各行に「ヘッダ」（frontmatter のみ）と「全体」（ファイル内容全体。agents=ファイル本体 / skills=SKILL.md。IPC `<agent>-asset:read-entry`）の 2 つの参照ボタンを 2 行で表示する。
- **設定の直接編集ダイアログ** — ダイアログを固定高（85vh）とし、スクロールはテキストエリア内の 1 本のみ（ネストスクロールなし）。
- **i18n** — 日本語/英語。共通文言は `locales/<lang>/app.ts`、agent 固有文言は `locales/<lang>/<agent>.ts` に分離し、`claude.*` / `codex.*` の名前空間で参照する。

## 4. 設定編集レジストリ

- Claude: `shared/agents/claude/constants.ts` の `SETTINGS_FIELDS`（`~/.claude/settings.json` の編集対象項目）。envFlag 型（env 内キーの ON/OFF）に対応。
- Codex: `shared/agents/codex/constants.ts` の `SETTINGS_FIELDS`（`~/.codex/config.toml` のドット区切りキー）。
- Grok: `shared/agents/grok/constants.ts` の `SETTINGS_FIELDS`（`~/.grok/config.toml` のドット区切りキー。公式 settings リファレンス準拠）。
- いずれも registry へ 1 項目追加すると読み取り・テーブル編集・保存・UI 表示まで反映される。関係のない既存項目には触れない差分マージ方式。

## 5. データモデル

本アプリはデータベースを使用しない。操作対象は各 CLI ツールの設定ファイル（JSON / TOML / YAML）とディレクトリのみで、独自の永続データは持たない。

## 6. agent 追加ガイド

新しい agent CLI（例: `gemini`）を追加する場合、以下の「agent 別ディレクトリの追加 + 登録ポイントへの 1 行追加」で完結する。

1. **shared** — `src/shared/agents/gemini/{constants,types,ipc}.ts` を作成。
   - 登録: `src/shared/ipc.ts` の `IpcApi` に `gemini: GeminiIpcApi;` を追加。
2. **main services** — `src/main/services/gemini/` にマネージャを実装。共通処理は `services/common/`（`WslDetector.getDistrosWithTest()` / `HomeFs`）と `main/utils/` を利用。
3. **main ipc** — `src/main/ipc/gemini/` にハンドラと `index.ts`（`registerGeminiIpcHandlers`）を作成。
   - 登録: `src/main/ipc/index.ts` に 1 行追加。
4. **preload** — `src/preload/agents/gemini.ts` にブリッジを作成。
   - 登録: `src/preload/index.ts` の api に `gemini: geminiApi,` を追加。
5. **renderer** — `src/renderer/agents/gemini/` に画面コンポーネントと `index.tsx`（`AgentModule` 定義: ルート/アイコン/文言キー）を作成。
   - 登録: `src/renderer/agents/index.ts` の `AGENT_MODULES` に 1 行追加。ダッシュボード・タイトルバー・ルートは自動反映される。
6. **i18n** — `src/renderer/i18n/locales/{ja,en}/gemini.ts` を作成し、各 `index.ts` に 1 行追加。文言キーは `gemini.*` 名前空間。

## 7. ビルド・配布

- 開発: `yarn dev`（tsc watch + vite + electron。renderer は BrowserRouter / <http://localhost:3001>）
- 検証: `yarn lint`（tsc --noEmit）/ `yarn eslint`
- 配布: `yarn dist:win|mac|linux`（renderer は HashRouter / `dist/renderer/index.html`）
- リリース: `yarn release:win|mac|linux`（GitHub ドラフトリリースへアップロード）

## 8. セキュリティ上の注意

- BrowserWindow は `contextIsolation: true` / `nodeIntegration: false` を維持しつつ、preload の agent 別ファイル分割のために `sandbox: false` としている。renderer に公開する API は `window.agentCliDevkit` 経由の明示的なブリッジのみ。
- クリーンアップ・削除操作はすべて既定 OFF のチェックボックス + 確認ダイアログを経る。使用中ファイルは best-effort でスキップし、結果として報告する。
